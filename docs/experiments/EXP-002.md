## Question

Can an immutable-baseline cumulative replay guard preserve the frozen 2%
public replay ceiling while retaining a path to the existing quantity gates?

EXP-001 answered the local version with a no-go. Its per-step guard accepted
all 200 candidates even though public replay regressed 2.685%. EXP-002 changes
only the authority at the optimizer boundary.

## Frozen authority

Before every candidate commits, the runtime evaluates one fixed validation
window from each of the six replay sources. It takes their arithmetic-mean loss
and compares it with the same composite evaluated under immutable ZERO.3. A
candidate is rolled back if the cumulative relative increase is non-finite or
greater than 1.5%.

The 1.5% hard limit is frozen before training. It leaves a 0.5
percentage-point reserve below the existing 2% balanced public replay ceiling
for fixed-window/evaluator mismatch. A 1% band is diagnostic only. The rotating
local delta, gradient cosine, and first-order drift remain diagnostics and have
no authority.

## Result

That risk materialized. Seed 2 committed its first 66 candidates. Attempts
67–74 each exceeded the frozen 1.5% ceiling and rolled back, so the declared
eight-rejection fallback ended the run. The largest accepted increase was
1.4253%; the largest rejected candidate reached 1.5463%.

The sentinel quantity gate still failed at commits 25 and 50. The run stopped
before its first 100-commit public evaluation, so there was no jointly feasible
checkpoint, promotion read, or replication opening.
