## Question

Can deterministic learning-rate backtracking preserve Q2.4's frozen cumulative
replay authority while reopening enough of the quantity-learning path to pass
the existing public and promotion gates?

EXP-002 showed that the 1.5% cumulative guard is a real safety boundary. It
committed 66 updates, then rolled back eight consecutive full-size candidates
before the first public checkpoint. EXP-003 changes candidate scale, not the
authority, task, data, or success criteria.

## Frozen intervention

One outer attempt freezes its minibatch, clipped gradient, proposed committed
update, weights, and both AdamW moment arrays. It evaluates learning-rate scales
`1`, `1/2`, `1/4`, `1/8`, `1/16`, `1/32`, `1/64`, and `1/128` in that order.
Before every retry, the full learned state is restored and the same moment
update is reapplied with only the learning rate scaled.

Every trial evaluates all six fixed replay slices against the immutable ZERO.3
composite. The first finite trial at or below the unchanged 1.5% cumulative
increase commits. If all eight fail, the outer attempt rejects atomically. Only
outer attempts consume the frozen 1000 acquisition and 400 consolidation
budgets. Eight consecutive exhausted outer attempts stop the phase.

The 2% public replay ceiling, quantity gates, 25-commit recovery cadence,
100-commit public cadence, promotion access rule, and replication lock are
unchanged. Projection remains disabled.
